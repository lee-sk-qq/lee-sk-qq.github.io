Types

Build goes through three steps:

  1. Preprocess : deals with source code meta data
  2. compile: turns source code into machine readable object files
  3. Link: Links objects into application

:: → scope resolution operator

Array

In CPP, it is better to use std::array as it can…

  • Know the accurate size
  • Can prevent certain bugs due to non-dynamic casting into a pointer
  • Easy to write an iterator.

The array size must be defined during compile step, hence size cannot be re-sized.

Vector

CPP library that provides “UNFIXED” size-container —> Dynamic Allocation

Vector is a generic container —> can put all kinds of types init.
